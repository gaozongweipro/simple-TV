# SimpleTV Android 直播应用设计规格

## 背景

SimpleTV 是一款面向小米小爱触屏音箱的安卓直播应用。目标设备性能较弱，屏幕分辨率约为 800x400，默认横屏使用，主要用于观看央视及卫视频道直播。

第一版聚焦“稳定、轻量、少操作”。应用不支持遥控器交互，不做节目单、账号、搜索、收藏、投屏等扩展功能。

## 目标

- 兼容 Android 8.0，最低 API 26。
- 默认横屏，适配 800x400 小屏。
- 使用触屏操作，不做遥控器焦点导航。
- 启动后尽快进入直播播放。
- 内置默认频道能力，同时支持远程更新频道清单。
- 播放源失效时自动尝试同频道备用源。
- UI 保持紧凑、美观，避免“老人机式”巨大控件。

## 非目标

- 不做 EPG 节目单。
- 不做账号系统。
- 不做收藏、历史、搜索。
- 不做弹幕、评论、投屏。
- 不做 WebView 播放器。
- 不要求用户在音箱上输入长地址。

## 技术路线

第一版采用原生 Android + Media3 ExoPlayer。

理由：

- Media3 ExoPlayer 对 HLS/m3u8 直播支持成熟。
- 原生 Android UI 开销低，更适合低性能设备。
- 播放失败监听、重试、备用源切换更容易做成可靠闭环。
- 相比 Flutter 或 WebView，运行时负担更小，兼容 Android 8 的风险更可控。

## 应用架构

应用使用单 Activity 结构。

主界面由两层组成：

- 播放层：Media3 ExoPlayer 的播放器视图，占满屏幕。
- 控制层：默认隐藏，点击播放画面后显示右侧频道抽屉和少量操作入口。

启动流程：

1. 强制横屏。
2. 读取本地缓存的频道清单和上次播放频道。
3. 如果存在可用缓存，优先开始播放，减少等待。
4. 后台请求固定远程 `channels.json`。
5. 远程清单更新成功后替换本地缓存。
6. 远程清单失败时保持当前缓存和播放状态。

## 频道清单

第一版使用 APK 内置固定远程 JSON 地址。音箱端不提供长地址输入界面。

建议 JSON 格式：

```json
{
  "version": 1,
  "updatedAt": "2026-05-15T00:00:00+08:00",
  "channels": [
    {
      "id": "cctv13",
      "name": "CCTV-13 新闻",
      "group": "央视",
      "sources": [
        {
          "url": "https://example.com/cctv13.m3u8",
          "label": "主源"
        },
        {
          "url": "https://example.com/cctv13-backup.m3u8",
          "label": "备用源"
        }
      ]
    }
  ]
}
```

本地状态只保存必要数据：

- 缓存的频道 JSON。
- 当前清单版本。
- 上次播放频道 ID。
- 短期失败源记录。

## 远程更新策略

频道维护通过固定远程 `channels.json` 完成。

- 应用启动时后台拉取远程清单。
- 用户可在设置面板中手动触发“刷新频道”。
- 拉取成功后校验 JSON 结构，再替换本地缓存。
- 拉取失败或校验失败时继续使用旧缓存。
- 如果完全没有可用缓存，显示“暂无频道清单”，只提供刷新操作。

## 播放失败策略

每个频道允许配置多个 HLS 源。

播放失败处理流程：

1. 监听 Media3 播放错误或加载超时。
2. 将当前源加入短期失败记录，例如 5 分钟内不再重试。
3. 自动尝试同频道下一个备用源。
4. 同频道所有源都失败后，显示轻量错误层。
5. 错误层提供“重试”和“切换频道”两个操作。

用户点击重试时，清除该频道的短期失败记录，并从第一个源开始重新尝试。

## 触屏交互

应用只面向触屏，不支持遥控器交互。

- 点击播放画面：显示或隐藏右侧频道抽屉。
- 频道抽屉宽度：屏幕约 1/3，即 800px 屏宽下约 266px。
- 点击频道：立即切换播放，抽屉自动收起。
- 点击抽屉外区域：收起抽屉。
- 长按播放画面：打开极简设置面板。
- 设置面板仅包含刷新频道、恢复默认、当前清单版本。

## 视觉与触控原则

第一版采用紧凑触控 UI。

- 可点击热区不小于 44dp。
- 文字视觉尺寸保持正常，不做超大字体。
- 频道行使用适度 padding 保证触控，而不是用巨大按钮占满屏幕。
- 频道抽屉使用半透明深色背景，减少对播放画面的干扰。
- 当前频道高亮清晰，但避免高饱和大色块。
- 800x400 下频道抽屉应尽量一屏显示 5 到 7 个频道。
- 设置入口不常驻占空间，避免干扰观看。

## 性能约束

- 单 Activity，不做复杂页面栈。
- UI 使用原生轻量 View，避免复杂嵌套和重动画。
- 播放器优先级最高，频道更新不能阻塞播放。
- 频道列表固定行高并复用列表项。
- 第一版不要求台标，避免额外图片加载和缓存开销。
- 启动阶段优先使用本地缓存尽快播放。
- 后台更新频道清单，更新成功后再替换缓存。

## 验收标准

- Android 8.0 设备或模拟器可安装运行。
- 800x400 横屏下无文字重叠、无整体滚动。
- 启动后能自动播放默认频道或上次频道。
- 点击播放画面能呼出或隐藏右侧频道抽屉。
- 频道抽屉在小屏上可触控，视觉不过度放大。
- 远程 `channels.json` 拉取失败时能继续使用本地缓存。
- 某个播放源失败时能自动切换到同频道备用源。
- 同频道所有源失败时显示轻量错误层并允许重试。
- App 无需登录、无需遥控器、无需用户在音箱上输入长地址。

## 后续实施边界

实施计划应优先完成以下闭环：

1. Android 工程骨架与横屏配置。
2. Media3 ExoPlayer HLS 播放。
3. 频道 JSON 模型、本地缓存和远程更新。
4. 右侧频道抽屉触屏交互。
5. 播放失败自动切备用源。
6. 800x400 布局与 Android 8 验证。

局域网管理页、二维码配置、EPG、台标和搜索均不进入第一版。
